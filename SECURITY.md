# Security Policy

## Supported versions

| Version | Supported |
|---|---|
| Latest release | Yes |
| Older releases | No — please upgrade |

## Reporting a vulnerability

**Do not open a public issue for security vulnerabilities.**

Send a report to **carlos.ksa21@gmail.com** with:

- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof of concept
- The version of Shrine you are using

You will receive a response within 72 hours. Once the issue is confirmed, a fix will be prepared and a patched release published before any public disclosure.
