# Contract: `docsgen` — Shrine CLI-reference generator

This contract defines the behavior of the auxiliary CLI-reference generator. It is the integration boundary between the Shrine Cobra command tree and the Hugo docs site: the docs build pipeline depends on this output shape, and any change here is a breaking change for the docs site.

## Architectural placement

**The generator is NOT a Shrine subcommand.** It lives in a separate Go module at:

```
docs/tools/docsgen/
├── go.mod                        # own module, with `replace` to the main repo
├── go.sum
├── docsgen.go                    # package docsgen
├── docsgen_test.go               # in-memory unit tests
└── cmd/
    └── docsgen/
        └── main.go               # package main, the binary
```

This isolation is required by the project memory note "Auxiliary tooling lives in a separate Go module": the main `shrine` binary's surface and `go.mod` MUST stay clean of docs-only concerns. The docsgen module imports `github.com/CarlosHPlata/shrine/cmd` (via local `replace`) and reads the Cobra tree through the public `cmd.RootCmd()` getter.

## Synopsis

```text
docsgen -out <output-dir> [-clean] [-include-hidden]
```

Invoked in practice via the project Makefile:

```text
make docs-gen-cli
```

which expands to:

```bash
cd docs/tools/docsgen && go run ./cmd/docsgen -out ../../content/cli -clean
```

## Description

Walks the Cobra command tree exposed by `cmd.RootCmd()` of the main Shrine module and writes one Markdown file per non-hidden command into `<output-dir>`, suitable for direct consumption by the Hugo docs site.

## Flags

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `-out` | string | (required) | Output directory. Created if missing. |
| `-clean` | bool | `false` | Remove existing `*.md` files in `<output-dir>` (excluding `_index.md`) before writing new ones. |
| `-include-hidden` | bool | `false` | Generate pages for hidden Cobra commands. Off by default. |

Flag style is the Go standard `flag` package (single-dash long flags), not Cobra/POSIX style. The tool is small, single-purpose, and not user-facing — it does not need a verb-noun command tree.

There is no `--dry-run` flag. The Constitution requires dry-run only for commands that mutate Docker or network state; this tool only writes files in the project's own working tree.

## Exit codes

| Code | Meaning |
|------|---------|
| 0 | Success — every applicable command produced a Markdown file. |
| 1 | Failure — output dir not writable, IO error, missing required flag, or generation error. Error written to stderr. |

## Output layout

For a Cobra tree with commands `shrine apply`, `shrine deploy`, `shrine status app`, `shrine status resource`:

```text
<output-dir>/
├── apply.md
├── deploy.md
├── status.md
├── status_app.md
└── status_resource.md
```

Subcommand pages use Cobra's standard underscore-joined naming, with the leading `shrine_` prefix stripped. The `_index.md` file (if present) is **never** overwritten — it is hand-curated.

## Generated file shape

Each generated file MUST start with Hugo YAML front-matter, followed by a "do not edit" banner, then the standard Cobra Markdown body:

```markdown
---
title: "apply"
description: "Apply manifests to state"
weight: 10
---

<!-- AUTO-GENERATED by `make docs-gen-cli` — DO NOT EDIT BY HAND. Edits will be overwritten on the next docs build. -->

## shrine apply

Apply manifests to state

### Synopsis
...

### Options
...

### SEE ALSO
...
```

`title` and `description` are derived from the Cobra command's `Use` (first token) and `Short` field respectively; double quotes inside the description are escaped (`\"`).

## Idempotency

Running `make docs-gen-cli` twice in a row, against the same source tree, MUST produce byte-identical output (modulo file timestamps). This is the property the CI drift check relies on:

```bash
cd docs/tools/docsgen && go run ./cmd/docsgen -out /tmp/expected
diff -r /tmp/expected ../../content/cli --exclude=_index.md   # MUST exit 0
```

`cobra/doc.GenMarkdownCustom` writes a timestamp footer by default; the tool sets `cmd.DisableAutoGenTag = true` (with `defer` to restore) before each render to suppress it.

## Behavior with hidden commands

By default, commands marked `Hidden: true` in Cobra are skipped, as is Cobra's auto-generated `help` command. `-include-hidden` reverses the Hidden filter (but `help` is always skipped).

## `_index.md` preservation

When `-clean` is set, the tool removes every `*.md` in the output directory **except** `_index.md`. The index page is always hand-curated and lives alongside the generated subcommand pages.

## Integration with the docs build

The docs build pipeline (`.github/workflows/docs.yml`, added in T028) runs:

```bash
make docs-gen-cli
cd docs && hugo --gc --minify --strict
```

A drift-check job (added in T041, US3) runs on every PR:

```bash
mkdir /tmp/expected-cli
cd docs/tools/docsgen && go run ./cmd/docsgen -out /tmp/expected-cli
diff -r /tmp/expected-cli ../../content/cli --exclude=_index.md
```

A non-empty diff means the committed CLI reference is out of sync with the current Cobra tree; the PR fails until the contributor runs `make docs-gen-cli` locally and commits the result.

## Test coverage

- **Unit** (`docs/tools/docsgen/docsgen_test.go`): in-memory generation against a synthetic Cobra tree using `bytes.Buffer` writers. Asserts front-matter, banner presence, hidden-command skip, `-include-hidden` reversal, `help` always skipped, YAML escaping for descriptions containing quotes. NO filesystem use, per the unit-tests-no-filesystem feedback memory.
- **End-to-end (drift check)**: the CI drift job (US3) is the de-facto integration test — running the real binary against the real Cobra tree and comparing to committed output exercises the full path. Constitution Principle V's `NewDockerSuite` integration gate does not apply here because the main `shrine` binary's behavior has not changed.
